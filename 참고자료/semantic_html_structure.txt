시맨틱 HTML 구조 정리 (학생·교사용)

1. 기본 구조 예시

<main>
  <aside>...</aside>
  <article>
    <section>...</section>
  </article>
</main>

2. 각 태그의 의미

- main : 페이지에서 가장 중요한 핵심 콘텐츠 영역
- aside : main 콘텐츠를 보조하는 영역 (사이드메뉴, 관련 정보 등)
- article : 독립적으로 완결되는 하나의 글 또는 콘텐츠 단위
- section : article(또는 문서) 안에서 주제별로 나눈 의미 있는 단락

3. 이 구조가 일반적인 이유

① HTML5 시맨틱 표준에 부합
   - main 안에 주 콘텐츠(article)와 보조 콘텐츠(aside)를 배치
   - article 안에서 section으로 글의 구조를 나눔

② 접근성과 검색엔진 최적화에 유리
   - 스크린 리더가 문서 구조를 명확히 인식
   - 검색엔진이 article 단위 콘텐츠를 잘 파악

③ CSS Grid/Flex 레이아웃과 궁합이 좋음
   - aside(좌), article(우) 구조를 만들기 쉬움

4. 언제 가장 적절한가?

다음 조건을 만족하면 매우 적절한 구조이다.
- 페이지의 중심이 ‘한 편의 글’일 때
- 글이 단독으로 공유 가능한 콘텐츠일 때
- aside가 본문을 보조하는 역할일 때
- section이 글의 소주제 역할일 때

5. 주의할 점

- 모든 페이지에 article을 사용할 필요는 없음
- 독립적인 글이 아니라면 section만 사용하는 것이 더 적절할 수 있음
- 의미 없는 묶음에는 section 대신 div 사용

6. 한 줄 요약

article : 혼자서도 이해되는 글
section : 그 글 안의 주제별 단락
aside   : 본문을 도와주는 보조 영역
main    : 페이지의 핵심 콘텐츠 영역
