--------------------------------
1. include.js / 공통 include 구조
--------------------------------
- header.html, menu.html, footer.html 등을 Ajax 방식으로 불러오는 스크립트 구조 설계.
- 상대 경로 문제로 인해 /header.html, /menu.html 을 찾지 못하던 오류 해결:
  → include.js 내 경로를 현재 프로젝트 트리 구조(HTML/common, HTML/menu, HTML/content 등)에 맞추어 수정.
- 각 메뉴 파일(menu-lectures.html, menu-webcolor.html 등)을 공통 include.js 한 곳에서만 경로 관리하도록 개선.

[핵심 포인트]
- 공통 경로 상수(PATH.commonDir, PATH.menuDir, PATH.contentDir 등) 사용.
- 개별 HTML에서 직접 경로를 수정하지 않고, JS 한 군데만 수정해도 전체가 반영되도록 구조화.

2. 탑 메뉴(Top Menu)와 드롭다운 서브메뉴
--------------------------------
- 상단 탑 메뉴에서 현재 선택된 항목 표시를 위해 .active 클래스 사용:

  .navbar a.active {
      color: red !important;
  }

- 드롭다운 서브메뉴 사이의 회색 구분선:

  #topMenu > ul > li.topMenuLi > ul.submenu > li {
      border-top: 1px solid #555;
  }

- 첫 번째 항목에는 선을 없애고 싶을 때:

  #topMenu > ul > li.topMenuLi > ul.submenu > li:first-child {
      border-top: none;
  }

- 상세 메뉴에서 hover 시 배경색/글자색 변경 처리.

3. 왼쪽 사이드 메뉴 공통 스타일 (6. 왼쪽 사이드 메뉴)
--------------------------------
- 사이드 메뉴 전체를 공통 스타일로 통일:

  #left-menu {
      width: 100%;
      background-color: #ffffff;
      border: none;
      padding: 0;
      min-height: 200px;
  }

  .side-menu {
      list-style: none;
      padding: 0;
      margin: 0;
      width: 100%;
      background: #2d2d2d;
      color: #ffffff;
      font-size: 1rem;
      font-weight: bold;
  }

- 모든 사이드 메뉴 항목 사이에 회색 구분선 적용:

  #left-menu .side-menu li + li {
      border-top: 1px solid #555555;
  }

- 사이드 메뉴 공통 링크 스타일:

  .side-menu a {
      display: block;
      padding: 8px 12px;
      text-decoration: none;
      color: #eeeeee;
  }

  .side-menu a:hover {
      background: #dddddd;
      color: red;
  }

4. DETAILILLUST 사이드 메뉴 전용 스타일 (6-1 최소 버전)
--------------------------------
- DETAILILLUST 페이지의 좌측 메뉴는 다른 메뉴들과 구조가 다르므로 전용 스타일 유지:

  .side-menu .side-section-title {
      background: #222222;
      color: #ffffff;

5. active 상태 자동 처리 (JS)
--------------------------------
- 탑 메뉴와 사이드 메뉴 모두 .active 클래스로 현재 선택 상태 표시.

- 사이드 메뉴 자동 active 예시:

  const sideMenu = document.querySelector(".side-menu, .side-menu-detail");
  if (sideMenu) {
      sideMenu.addEventListener("click", (e) => {
          const link = e.target.closest("a.side-link[data-content]");
          if (!link || !sideMenu.contains(link)) return;
          e.preventDefault();

          let url = link.getAttribute("data-content");
          if (!url) return;
          if (!url.includes("/")) {
              url = PATH.contentDir + url;
          }

          loadHtmlInto("#main-content", url);

          sideMenu.querySelectorAll("a.side-link").forEach((a) => {
              a.classList.remove("active");
          });
          link.classList.add("active");
      });

      if (!sideMenu.querySelector("a.side-link.active")) {
          const firstLink = sideMenu.querySelector("a.side-link");
          if (firstLink) firstLink.classList.add("active");
      }
  }

7. DETAILILLUST 레이아웃 및 반응형
--------------------------------
- PC 화면에서: 왼쪽 퀵 메뉴 + 오른쪽 본문 2열 레이아웃:

  .detailillust-layout {
      position: relative;
      display: grid;
      grid-template-columns: 260px minmax(0, 1fr);
      gap: 10px;
  }

  #quick {
      position: absolute;
      left: -150px;
      top: -20px;
  }

- 1024px 이하(태블릿/모바일)에서 1열로 변경:

  @media (max-width: 1024px) {
      .detailillust-layout {
          grid-template-columns: 1fr;
      }

      #quick {
          position: static;
          top: auto;
          left: auto;
          margin-bottom: 10px;
          width: 100%;
      }

      #detail-contents,
      .detailillust-article {
          grid-column: 1 / 2;
      }
  }

- 내용 전용 파일(content-detailillust-02.html, -03.html)은
  .detailillust-layout과 #quick이 없으므로 기본 1열 본문 페이지처럼 동작.

8. border 관련 질문 요약
--------------------------------
- #left-menu에 border: 1px solid #aa0000; 가 있으면 상하좌우 테두리가 모두 생김.
  → 외곽선이 필요 없으면 border: none; 또는 border-right만 남기기.

- .side-link 사이 회색선:
  - li + li { border-top: 1px solid #555; } 규칙으로 "항목 사이에만" 선을 표시.
  - 마지막 항목 아래선 제거는 li:last-child 에서 border-bottom: none; 등으로 처리.

- 테스트용 border, background-color는 최종 CSS에서 제거.
