[📌 웹 프로젝트 파일 기반 문서 정리]

이 문서는 업로드한 HTML / CSS / JS 파일들을 기준으로
프로젝트 구조와 역할을 정리한 문서입니다.

--------------------------------------------
📍 1. 프로젝트 목적
--------------------------------------------
- 공통 header / footer / 메뉴를 include.js로 로드하는 구조
- 콘텐츠 파일을 페이지 본문 영역에 동적 로딩
- 주제별 페이지 분리 (WEB COLOR / DETAILILLUST / LECTURES 등)

--------------------------------------------
📍 2. 프로젝트 디렉터리 구조
--------------------------------------------

project/
 ├─ index.html                ← 사이트 메인 페이지
 ├─ header.html               ← 공통 상단 영역
 ├─ footer.html               ← 공통 하단 영역
 ├─ include.js                ← HTML include + 메뉴/본문 로딩 담당
 ├─ style.css                 ← 전체 사이트 스타일
 │
 ├─ menu/                     ← 상단·사이드 메뉴 HTML
 │   ├─ menu.html
 │   ├─ menu-about.html
 │   ├─ menu-webcolor.html
 │   ├─ menu-detailillust.html
 │
 ├─ content/                  ← 페이지별 콘텐츠 파일
 │   ├─ content-about.html
 │   ├─ content-webcolor.html
 │   ├─ content-lectures.html
 │   ├─ content-tagcloud.html
 │   ├─ content-detailillust.html          ← ✔ 대표 DETAILILLUST
 │   ├─ content-detailillust-02.html       ← ✔ 세부 페이지
 │   ├─ content-detailillust-03.html       ← ✔ 세부 페이지
 │
 └─ assets/ (없지만 향후 추가 권장)
     ├─ img/
     ├─ css/
     ├─ js/

--------------------------------------------
📍 3. 각 파일 역할 정리
--------------------------------------------

● index.html
- 메인 entry point
- include.js가 이 파일의 영역에 header/menu/footer를 삽입

● header.html / footer.html
- 공통 레이아웃
- 모든 페이지 로딩 시 유지

● include.js
- HTML include 기능
- 메뉴 active 적용
- content/*.html 로딩 제어 중심

● style.css
- 전체 페이지 공통 스타일
- DETAILILLUST 전용 스타일 포함

● menu/*.html
- 상단 메뉴 또는 사이드 메뉴 UI 구조
- 데이터는 HTML 기반, 로직은 include.js 담당

● content/*.html
- 본문 내용 조각 파일
- index.html or include.js에 의해 로드됨

--------------------------------------------
📍 4. DETAILILLUST 페이지 구조 요약
--------------------------------------------

● content-detailillust.html
- 2단 구조: 왼쪽 Quick 메뉴 + 오른쪽 본문

● content-detailillust-02.html / -03.html
- 본문 단독 페이지 (일반 페이지처럼 표시됨)
- grid layout 없음 → 자연스럽게 단일 레이아웃

--------------------------------------------
📍 5. 구조적 장점
--------------------------------------------

- HTML 콘텐츠를 작은 모듈로 분리 → 유지보수 용이
- include.js에서 경로 변경 시 전체 반영
- 메뉴 및 본문 데이터 확장 용이
- 반응형 디자인 적용 기반 준비 완료

--------------------------------------------
📍 6. 향후 개선 제안
--------------------------------------------

(1) assets/js, assets/css 디렉터리 분리  
(2) style.css를 페이지/기능별로 분리  
(3) 이미지 자산을 img 폴더로 일원화  
(4) 브라우저 캐시 기반 성능 최적화  
(5) GitHub Pages 배포 구조 정리 가능
